TYPE: ROOT-CAUSE FOUND + FIXED (after a long wrong-layer detour — read this so you don't repeat it).

THE ACTUAL ROOT CAUSE (the human found it: "it happens in FLY mode too, just by looking UP — the ground disappears"): world_view._make_page_instance set each page's custom_aabb vertical extent from the OLD amplitude (240) — `AABB(-amplitude .. 3*amplitude)`. The terrain mesh is a flat plane displaced in the VERTEX SHADER, so Godot can't auto-compute its bounds and uses that custom AABB for frustum culling. The composition terrain reaches ~+1.9km / ~-1km — ~8x outside the AABB. So tilting the camera moved the (too-short) AABB out of the frustum and Godot CULLED the page -> terrain vanished at certain angles. That angle-dependent vanish read as "clipping/seeing under the ground."
EVIDENCE THAT REDIRECTED ME: a probe measured the collision surface == terrain cell heights within 4 cm (collision was CORRECT all along); and m1_7c always passed. The "low collision" was a false lead. The human's fly-mode-look-up observation was decisive — a camera-angle-dependent render symptom can't be collision.
FIX: world_view const AABB_HALF_HEIGHT=4000; page AABB now -4000..+4000 vertical. Oversized AABB only costs a little over-draw, never a wrong cull. Human confirmed: terrain stays visible at all angles. m1_7c still PASS (render-only change).
LESSON (burned in): a camera-ANGLE-dependent visual symptom is a RENDER bug (culling/AABB/LOD), not collision/physics. Check the render layer FIRST when "ground disappears" depends on where you look. I chased the character controller for too long; the systematic measurement (collision is fine) + the human's angle clue is what cracked it. Also: vertex-shader-displaced meshes ALWAYS need a custom AABB that covers the real displacement range — revisit AABB_HALF_HEIGHT if terrain ever exceeds ~4km.
